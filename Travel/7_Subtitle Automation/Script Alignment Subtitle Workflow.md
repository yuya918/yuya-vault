# 旅動画 字幕作成ワークフロー手順書

海外旅行YouTubeロング動画の字幕を、台本(スクリプト)照合方式で高精度に自動化するための手順書。

---

## このワークフローの肝

**テキストは台本から / タイムコードは音声解析JSONから**

- 録音の元になったChatGPT台本(正しいテキスト)と、DaVinci Resolve等が出力した単語レベルのタイムスタンプ付きJSON(正しい時刻)を照合する
- テキストは台本由来なので**誤認識ゼロ**(例: 重慶→従兄、洪崖洞→本屋、火鍋→日米 のような音声誤認識が起きない)
- 時刻はJSONの各単語の実データ由来なので**ズレゼロ**(按分・推定が不要)
- これは forced alignment(強制アラインメント)と呼ばれる確立した手法

---

## 全体の流れ

```text
ChatGPTで台本を作成(録音の元原稿)
  → その台本を読んでナレーション録音
  → 編集を確定
  → DaVinci Resolve等で単語レベルのタイムスタンプ付きJSONを書き出し
  → 【台本 + JSON の2つ】をClaudeに渡す
  → 日本語のみ / 英日2点セット を確認
  → 台本テキスト × JSON時刻でSRT生成
  → SRTをPremiere/DaVinciに読み込み
  → 書き出し
```

---

## Claudeへの依頼ルール(重要)

字幕作成を依頼するときは、**必ずChatGPTで作った台本(録音の元原稿)を一緒に渡す**こと。台本がないと音声誤認識を直せず、精度が落ちる。

依頼時にClaudeが守るルール:

1. **台本テキストを正とする** — 表示テキストは台本から取る
2. **言い回しが違う場合は音声(JSON)を信用する** — 録音時に台本から少し変えて喋ることがあるため。例: 台本「〜ありました」→ 実際は「〜でした」「〜だった」、あるいは何も言っていない等。この場合は音声側に合わせる
3. **タイムコードはJSONの単語時刻から取る**(按分しない)
4. **1行は約15文字、必ず1行のみ**(2行にしない)、意味のまとまりで分割
5. **文末の句読点・句点は不要**(日本語・英語とも)。文中の区切り(例: 北京、上海、深圳)は残してよい
6. **日本語のみか英日2点セットかを毎回確認する**(動画によって異なるため)

### プロンプトテンプレート(新しい会話で使う場合)

> 字幕SRTを作ってください。台本JSONと、ChatGPTで作った台本テキストを添付します。
> ・テキストは台本を正とする。ただし台本と言い回しが違う箇所は音声(JSON)を信用する(〜でした/〜だった/言っていない等)
> ・タイムコードはJSONの単語時刻から取る(按分しない)
> ・1行は約15文字、必ず1行のみ、意味のまとまりで分割
> ・文末の句読点・句点は不要(日英とも)
> ・日本語のみ / 英日2点セット → ★毎回ここを確認

## 文字起こしJSONの書き出し(DaVinci Resolve)

- DaVinci Resolveの文字起こし機能で、単語レベルのタイムスタンプ付きJSONを書き出す
- 各単語に start / duration / confidence(信頼度)を持つ
- confidenceが低い単語(目安0.5未満)は誤認識の可能性が高いが、台本照合方式なら台本側が正なので影響を受けない
- DaVinci経路はPremiereの「言語パックダウンロードエラー」とも無縁

### Premiere経路(代替)

- Premiere Proの自動文字起こし → SRT書き出しでも可能だが、SRTは文単位の時刻しか持たないため、文を分割する際に時刻を文字数比で按分する必要があり、ズレが出やすい
- 言語パックエラーが出る場合: Creative Cloud Desktop → Premiere Pro の「…」→ アドオンから日本語を手動追加するのが最有力。それでも不可なら Creative Cloud 経由で起動／セキュリティソフト・VPNの通信許可確認／環境設定リセット(起動時 option キー)

## SRT整形でClaudeが行う処理

| 処理 | 内容 |
|---|---|
| テキスト確定 | 台本を正として表示テキストを作る |
| 言い回し差異の処理 | 台本と違う箇所は音声を信用(〜でした/〜だった/無言など) |
| 時刻割り当て | 各行の開始・終了をJSONの単語時刻から取得(按分なし) |
| 1行15文字分割 | 長い行は意味単位で複数ブロックに分割、時刻も単語境界で分ける |
| 句読点除去 | 文末の。．、！？等を削除(文中の区切りは残す) |

---

## タイムコードのズレが出たときの対処(SRT経路の場合)

| パターン | 症状 | 対処 |
|---|---|---|
| A: 一定 / 線形 | 全体が同じ秒数ズレ、または後半ほど広がる(fps不一致) | 正しい時刻のアンカー2点を渡せば一括線形補正 |
| B: 不規則 | 合う箇所とズレる箇所が混在 | SRT側では修正困難。①最終確定シーケンスに文字起こし ②Subtitle Edit のポイント同期 |

※ 台本照合方式(JSON使用)なら、そもそもこのズレ問題は構造的に発生しない。

## Claudeにできないこと

- 音声を聴くことはできない(JSONや台本などテキスト/数値データが必要)
- 長尺MP4はアップロード容量の制限を超える
